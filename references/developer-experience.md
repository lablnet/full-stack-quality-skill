# Developer Experience Standards

Use for local setup, scripts, onboarding, debugging, seed data, test data, and day-to-day maintainability.

- A new developer or agent should be able to install, run, test, and debug the project from documented commands.
- Keep scripts discoverable and consistent.
- Provide safe seed data and local environment examples.
- Document required services, ports, env vars, and common troubleshooting.
- Avoid hidden manual steps.

Review smells: README setup fails, missing env var docs, only one person knows how to run workers, tests require hidden services, seed data is production-derived.
