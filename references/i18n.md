# Internationalization Standards

Use when a project supports or may soon support multiple locales.

- Extract user-facing strings.
- Format dates, times, numbers, currencies, names, addresses, and plurals with locale-aware APIs.
- Avoid hardcoded text in components, templates, validation, emails, and errors.
- Support RTL layout when required.
- Do not concatenate translated sentence fragments unless the i18n system supports it safely.

Review smells: hardcoded strings, manual date formatting, string concatenation for plurals, layout assumes English text length, missing translation keys.
