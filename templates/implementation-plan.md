# Implementation Plan Template

Use this before broad edits or multi-layer changes.

## Scope

What will change and what will stay untouched.

## Steps

1. Database changes, if any.
2. Backend changes, if any.
3. HTTP API changes, if any.
4. Frontend changes, if any.
5. Tests and validation.

## Risks

- Data integrity, auth, migration, compatibility, or UI risks.

## Validation

- Commands or manual checks to run.
